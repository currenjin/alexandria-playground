### Q1. 관심사 분리와 모듈성의 차이는?
A. 모듈성은 "독립 변경 가능한 단위로 나누는 것"이고, 관심사 분리는 "무엇을 기준으로 나눌 것인가"에 대한 답이다. 관심사 분리는 모듈 경계를 정하는 가이드라인이다.

### Q2. 포트와 어댑터 아키텍처의 핵심 이점은?
A. 비즈니스 로직이 어떤 인프라에도 의존하지 않아, 인프라를 교체해도 비즈니스 로직이 변하지 않는다. 테스트 시 실제 DB 대신 인메모리 구현을 쓸 수 있어 테스트 속도도 향상된다.

### Q3. "수직 슬라이스"와 "기능 기반 패키지"의 관계는?
A. 수직 슬라이스는 아키텍처 패턴이고, 기능 기반 패키지는 코드 조직 방법이다. 기능 기반 패키지가 수직 슬라이스를 코드 레벨에서 구현한 것이라고 볼 수 있다.

### Q4. 관심사를 과도하게 분리하면?
A. 모듈 간 통신이 복잡해지고, 간단한 기능 추가에도 여러 모듈을 거쳐야 한다. "이 수준의 분리가 실제로 독립적 변경을 가능하게 하는가?"를 기준으로 판단해야 한다.

### Q5. Bounded Context 간 데이터 공유 방법은?
A. (1) 동기 API 호출: 단순하지만 결합도 증가. (2) 이벤트/메시지: 느슨한 결합, 최종 일관성. (3) 공유 DB: 피해야 함, 스키마 변경이 모든 컨텍스트에 영향. (4) 데이터 복제: 각 컨텍스트가 필요한 데이터의 복사본을 유지.
