# Release It! (2nd) - Chapter 01 Notes

## 핵심 메시지
- 기능 자체보다 **운영 중 실패 모드(failure mode)** 를 먼저 고려해야 함.
- “잘 동작하는 코드”와 “운영에서 살아남는 코드”는 다름.

## 실무 체크리스트
- 타임아웃 기본값이 명시되어 있는가?
- 재시도 정책에 backoff/jitter가 있는가?
- 외부 의존성 실패 시 graceful degradation이 가능한가?
- 장애 시 관측 가능한 지표/로그/트레이스가 있는가?

## 액션 아이템
1. 서비스별 외부 호출 타임아웃 표준화
2. 장애 시나리오 1개(GameDay) 리허설
3. Runbook 초안 작성 (탐지 → 완화 → 복구)
