# DDIA Phase Mission Pack

DDIA 학습을 "읽기"에서 끝내지 않고, 실제 설계/구현/장애대응까지 연결하기 위한 미션 팩입니다.

## 진행 규칙
1. 미션 단위로 진행합니다. (`읽기 -> 구현 -> 실패 재현 -> 측정 -> 회고`)
2. 각 미션 시간 박스는 `4~6시간`입니다.
3. 각 미션 완료 시 아래 3개를 반드시 남깁니다.
   - 코드/설정 변경
   - 실험 로그 (측정치 포함)
   - 1페이지 회고

## 권장 순서
1. [Phase 1 - Foundations](./phase-1-foundations.md)
2. [Phase 2 - Reliability](./phase-2-reliability.md)
3. [Phase 3 - Distributed Workflows](./phase-3-distributed-workflows.md)
4. [Phase 4 - Streaming and Decisions](./phase-4-streaming-and-decisions.md)

## 공통 완료 기준
- 미션마다 "완료 조건"을 수치로 기록합니다. (예: P95, 오류율, lag)
- 실패 케이스를 최소 1개 이상 재현합니다.
- 회고에 "무엇이 깨졌는지/어떻게 검증했는지"를 명확히 적습니다.
