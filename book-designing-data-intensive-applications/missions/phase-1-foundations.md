# Phase 1: Foundations

목표: 데이터 모델, 인덱스, 스키마 진화의 기초 의사결정을 직접 검증한다.

## Mission 1.1 - 모델링 비교 실험
- 시나리오: 상품/주문/재고 기능을 빠르게 만들어야 한다.
- 작업:
  - 관계형 모델(Postgres) 스키마 설계
  - 문서형 모델(문서 DB 가정) 스키마 설계 문서화
  - 동일 기능 쿼리 5개(상품 조회, 주문 생성, 재고 차감, 주문 내역, 인기 상품) 정의
  - 두 모델의 변경 비용/조인 복잡도/중복 데이터 리스크 비교표 작성
- 완료 조건:
  - 쿼리 5개에 대해 실행 계획 또는 예상 접근 경로를 설명
  - "왜 이 서비스에서는 관계형(또는 문서형)을 우선하는지" 5문장 내 결정 근거 작성
- 제출물:
  - 스키마 DDL 또는 모델 다이어그램
  - 비교표 1개
  - Mission Log

## Mission 1.2 - 인덱스 튜닝
- 시나리오: 상품 검색 API가 느리다.
- 작업:
  - 기준 쿼리 3개 선정
  - 인덱스 적용 전/후 벤치마크
  - 커버링 인덱스 또는 복합 인덱스 1개 적용
  - 쓰기 성능 저하(INSERT/UPDATE) 영향도 측정
- 완료 조건:
  - 최소 1개 쿼리에서 `P95 30% 이상` 개선
  - 쓰기 성능 저하율을 수치로 기록하고 허용 여부 판단
- 제출물:
  - 벤치마크 로그
  - 인덱스 설계 근거
  - Mission Log

## Mission 1.3 - 스키마 진화
- 시나리오: 주문 이벤트 스키마를 v1 -> v2로 변경해야 한다.
- 작업:
  - 이벤트 스키마(v1, v2) 정의
  - 변경 유형 2개 포함 (필드 추가, optional/default 처리)
  - 구버전 소비자 호환성 테스트
  - 배포 순서(Producer/Consumer) 시뮬레이션
- 완료 조건:
  - 구버전 소비자에서 역직렬화 실패 0건
  - 롤백 시나리오 1개 문서화
- 제출물:
  - 스키마 파일(v1, v2)
  - 호환성 테스트 결과
  - Mission Log
