# plan.md - Lotto Game (Kotlin)

## Phase 1: 로또 번호 (LottoNumber)

### 로또 번호 생성
- [x] 1~45 범위의 숫자로 LottoNumber 생성
- [x] 0 이하의 숫자로 생성 시 예외 발생
- [x] 46 이상의 숫자로 생성 시 예외 발생

### 로또 번호 동등성
- [x] 같은 숫자의 LottoNumber는 동등함
- [ ] 다른 숫자의 LottoNumber는 동등하지 않음

---

## Phase 2: 로또 한 장 (Lotto)

### 로또 생성
- [ ] 6개의 LottoNumber로 Lotto 생성
- [ ] 6개 미만의 숫자로 생성 시 예외 발생
- [ ] 6개 초과의 숫자로 생성 시 예외 발생
- [ ] 중복된 숫자가 있으면 예외 발생

### 로또 번호 포함 여부
- [ ] 특정 LottoNumber가 Lotto에 포함되어 있는지 확인

### 로또 번호 일치 개수
- [ ] 다른 Lotto와 일치하는 번호 개수 반환

---

## Phase 3: 로또 생성기 (LottoGenerator)

### 자동 생성
- [ ] 1~45 중 랜덤한 6개 숫자로 Lotto 생성
- [ ] 생성된 Lotto의 번호는 중복되지 않음

### 수동 생성
- [ ] 입력받은 숫자들로 Lotto 생성

---

## Phase 4: 당첨 번호 (WinningLotto)

### 당첨 번호 생성
- [ ] 당첨 번호 6개와 보너스 번호 1개로 WinningLotto 생성
- [ ] 보너스 번호가 당첨 번호와 중복되면 예외 발생

### 당첨 결과 확인
- [ ] Lotto와 비교하여 일치 개수 반환
- [ ] 보너스 번호 일치 여부 반환

---

## Phase 5: 당첨 등수 (Rank)

### 등수 정의
- [ ] 6개 일치 -> 1등 (2,000,000,000원)
- [ ] 5개 일치 + 보너스 일치 -> 2등 (30,000,000원)
- [ ] 5개 일치 -> 3등 (1,500,000원)
- [ ] 4개 일치 -> 4등 (50,000원)
- [ ] 3개 일치 -> 5등 (5,000원)
- [ ] 2개 이하 일치 -> 낙첨 (0원)

### 등수 판정
- [ ] 일치 개수와 보너스 일치 여부로 Rank 결정

---

## Phase 6: 로또 구매 (LottoStore)

### 구매 금액 검증
- [ ] 1,000원 단위로만 구매 가능
- [ ] 1,000원 미만 구매 시 예외 발생
- [ ] 1,000원 단위가 아니면 예외 발생

### 로또 구매
- [ ] 구매 금액에 해당하는 만큼 Lotto 발급
- [ ] 1,000원 -> 1장, 5,000원 -> 5장

---

## Phase 7: 당첨 통계 (LottoResult)

### 통계 생성
- [ ] 구매한 Lotto 목록과 WinningLotto로 통계 생성
- [ ] 각 등수별 당첨 개수 집계

### 수익률 계산
- [ ] 총 당첨 금액 / 구매 금액 * 100 = 수익률(%)
- [ ] 소수점 둘째 자리까지 표시

---

## Phase 8: 입출력 (View)

### 입력
- [ ] 구매 금액 입력
- [ ] 당첨 번호 입력 (쉼표로 구분)
- [ ] 보너스 번호 입력

### 출력
- [ ] 구매한 로또 목록 출력
- [ ] 당첨 통계 출력
- [ ] 수익률 출력

---

## Phase 9: 컨트롤러 (LottoController)

### 게임 실행
- [ ] 전체 게임 흐름 통합
- [ ] 입력 -> 구매 -> 당첨 확인 -> 통계 출력
