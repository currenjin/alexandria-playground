# 객체지향의 사실과 오해
> 역할, 책임, 협력 관점에서 본 객체지향

- 저자: 조영호
- 출판: 위키북스

## 핵심 메시지
객체지향의 핵심은 **클래스가 아니라 객체**다. 객체지향은 **자율적인 객체들의 협력**으로 애플리케이션을 구축하는 것이며, 실세계의 모방이 아닌 **새로운 세계의 창조**다.

## 핵심 개념

### 객체의 세 가지 요소
- **상태(State)**: 객체의 특정 시점의 조건이나 상황
- **행동(Behavior)**: 외부 요청에 응답하고 상태를 변경하는 활동
- **식별자(Identity)**: 객체를 구별하는 고유한 속성

### 책임의 두 가지 범주
- **아는 것(Knowing)**: 사적인 정보, 관련 객체, 유도할 수 있는 것
- **하는 것(Doing)**: 스스로 하기, 다른 객체 시작시키기, 활동 제어하기

### 세 가지 설계 관점
- **개념 관점**: 도메인 개념과 관계
- **명세 관점**: 객체의 인터페이스 (무엇을)
- **구현 관점**: 인터페이스 구현 (어떻게)
