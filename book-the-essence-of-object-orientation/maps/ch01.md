# Ch01. 협력하는 객체들의 공동체

## TL;DR
- 객체지향의 핵심은 **클래스가 아니라 객체**다.
- 객체지향은 **자율적인 객체들의 협력**으로 애플리케이션을 구축하는 것이다.
- 객체는 **역할, 책임, 협력**을 기반으로 행동한다.
- 실세계의 모방이 아닌 **새로운 세계의 창조**가 객체지향이다.

## Key Ideas
- **커피 공화국 메타포**: 손님, 캐셔, 바리스타가 협력하여 커피를 만드는 과정이 객체지향의 본질을 설명한다.
- **역할(Role)**: 협력 내에서 특정 위치를 차지하는 객체의 책임 집합이다.
- **책임(Responsibility)**: 객체가 협력에 참여하기 위해 수행하는 행동이다.
- **협력(Collaboration)**: 요청과 응답의 연쇄로 이루어진 객체들의 상호작용이다.
- **자율성(Autonomy)**: 객체는 스스로 판단하고 결정할 수 있어야 한다.
- **메시지(Message)**: 객체 간 유일한 소통 수단이다. 어떻게가 아닌 무엇을 요청한다.
- **메서드(Method)**: 메시지를 수신한 객체가 요청을 처리하는 방법이다.

## Trade-offs

| 관점 | 장점 | 단점 | 언제 |
|------|------|------|------|
| 실세계 메타포 | 이해하기 쉽고, 도메인 모델링에 유용하다 | 소프트웨어 객체는 능동적이라 차이가 있다 | 초기 설계 시 개념 이해에 활용한다 |
| 역할 기반 설계 | 유연성이 높고 대체가 쉽다 | 과도한 추상화 위험이 있다 | 변경 가능성이 높을 때 적용한다 |
