# PR Description Generator

현재 작업한 내용을 기반으로 PR 설명을 생성합니다.

## 출력 형식

```markdown
# Problem
> 해결해야 하는 문제가 무엇인가요?

# Solution
> 그 문제는 어떻게 해결했나요?

# Etc
> 기타 하고싶은 말이 있다면 적어주세요!
```

## 작성 스타일

- **간결하게**: 불필요한 설명 없이 핵심만
- **bullet point 사용**: 문장형보다 bullet point 위주로
- **"~했습니다" 형식**: 짧고 명확하게

## 예시

### 예시 1
```markdown
# Problem
> 해결해야 하는 문제가 무엇인가요?
- 배차최적화 시 차량의 차고지를 가져오는 규칙이 변경되어야 합니다. 순서는 다음과 같음.
  - 1) 차량의 차고지
  - 2) 거래처의 차고지
  - 3) 조직의 차고지
  - 4) 기본 차고지(신탄진로 1)

# Solution
> 그 문제는 어떻게 해결했나요?
- PreOptimization에서 차량의 차고지를 추가하도록 수정했습니다.
- 존재하지 않는 경우 기본 차고지를 가져오도록 했습니다.
- 이를 위한 Reader, Domain 함수가 추가되었습니다.
```

### 예시 2
```markdown
# Problem
> 해결해야 하는 문제가 무엇인가요?
- 운임파일 변경으로 운임표를 변경후 오더상세에서 운임을 조회하면 신규 운임표로 매출가를 조회하지않습니다.
- 엑셀로 오더 등록시 매출가 계산이 불가능한 상황에 오렌지 경고를 제공하지않습니다.
- 차량단위 필터에 차량단위 순서 확인부탁드립니다.

# Solution
> 그 문제는 어떻게 해결했나요?
- deletedAt is null 처리
- 매출액이 없는 경우 null 처리
- 차량 단위 순서 order 수정
```

## 실행

1. `git diff`, `git log`로 변경사항 분석
2. 위 스타일에 맞춰 간결하게 PR 설명 생성
3. 복사 가능한 코드 블록으로 출력
