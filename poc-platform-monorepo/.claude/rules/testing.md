# 테스트 규칙

- 두 층으로 나눈다(BE):
  - **단위** — 도메인·백본 순수 로직, 외부 의존 없음, 빠름 (JUnit5 + Mockito). 예: 애그리거트 가드 전이, 사가 반응, Outbox/Inbox·릴레이 로직.
  - **통합(IT)** — DB·Kafka 실제 왕복 (Testcontainers). `Outbox → 릴레이 → Kafka → 소비 → Inbox` 왕복·멱등·순서·보상·상태 전이.
- **커버리지 %가 목표가 아니라 핵심 경로를 필수로 덮는다** — 골든패스, 상태 전이, 정합성(멱등·보상·DLT).
- **의미 있는 케이스**를 쓴다: 정상 경로 + 엣지(빈 배치·null·중복 헤더) + 실패/보상(역직렬화→DLT · 발행실패→순서보존 · 거부→보상) + 멱등(중복 skip). **게터·설정 테스트나 성급한 100% 강박은 지양** — 방어적 도달 불가 코드는 남겨도 된다.
- 새 이벤트·핸들러·사가를 추가하면 **그에 맞는 테스트를 같이 추가**한다.
- (기계 강제) CI에서 테스트 통과 없이 머지 금지. 레이어 의존은 ArchUnit으로 강제 가능(후속).
