# Ch31. Coupling Versus Decoupling

## TL;DR
- 결합을 줄이는 것(Decoupling)에도 비용이 든다.
- 모든 결합을 제거하려 하지 말고, 변경이 잦은 곳에 집중하라.

## Key Ideas
- Decoupling 비용: 인터페이스 추가, 간접층 증가, 복잡성 증가
- 변경이 없는 곳의 결합은 문제가 아니다
- 자주 변경되는 곳의 결합이 진짜 문제
- 정리 우선순위 = 변경 빈도 × 결합도
